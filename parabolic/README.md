# Parabolic - AI Day Planner (Convex + LangChain JS)

This is a new implementation of the AI Day Planner using **Convex** as the backend and **LangChain JS** for the AI agent. It replaces the Python FastAPI + PostgreSQL stack with a unified TypeScript solution.

## Architecture


## Key Features

- **Streaming Chat**: Real-time AI responses using Convex HTTP Actions
- **Semantic Search**: Vector-based todo search using Convex's built-in vector indexing
- **AI Suggestions**: LangChain JS agent suggests todos with accept/reject workflow
- **Day Math**: Date calculations for planning
- **Todo CRUD**: Full todo management with auto-generated embeddings

## Prerequisites

1. **Node.js** 18+ and npm
2. **Convex account** - Sign up at [convex.dev](https://convex.dev)
3. **Ollama** running locally with:
   - `qwen3:latest:latest` model for chat
   - `nomic-embed-text-v2-moe:latest` model for embeddings

## Setup

### 1. Install Dependencies

```bash
cd parabolic
npm install
```

### 2. Initialize Convex

```bash
npx convex dev
```

This will:
- Set up a Convex project
- Start the Convex dev server
- Generate the `_generated` folder with types

### 3. Environment Variables

Create `.env.local`:

```env
NEXT_PUBLIC_CONVEX_URL=your_convex_deployment_url
OLLAMA_BASE_URL=http://host.docker.internal:11434
```

For local development with Ollama on macOS/Windows, use `host.docker.internal`. For Linux, you may need to use `localhost` or set up a tunnel.

### 4. Start Development Server

In one terminal (Convex is already running from step 2):

```bash
npm run dev
```

The app will be available at `http://localhost:3000`.

## Comparison with Original Stack

| Feature         | Original (Python)            | New (Convex + LangChain JS)    |
| --------------- | ---------------------------- | ------------------------------ |
| Backend         | FastAPI + PostgreSQL + SQLAlchemy | Convex (serverless)      |
| Vector Search   | PGVector                     | Convex vectorSearch            |
| AI Agent        | LangChain Python             | LangChain JS                   |
| Streaming       | Python SSE                   | Convex HTTP Actions            |
| Embeddings      | OllamaEmbeddings (Python)    | Ollama HTTP API (JS)           |
| Database Migrations | Alembic                | Convex schema (automatic)      |
| Frontend        | Next.js + Axios              | Next.js + Convex React         |

## Agent Tools

The AI agent has access to these tools:

1. **get_todos** - Retrieve all current todos
2. **search_todos** - Semantic search for todos
3. **day_math** - Date calculations (add/subtract days, etc.)
4. **suggest_todo** - Create a pending suggestion for user approval

## Development

### Adding New Tools

Edit `convex/agent.ts` to add new LangChain tools:

```typescript
const myNewTool = tool(
  async (args) => {
    // Tool implementation
    return result;
  },
  {
    name: "my_tool",
    description: "What this tool does",
    schema: z.object({
      param: z.string(),
    }),
  }
);
```

### Schema Changes

Edit `convex/schema.ts` and the dev server will automatically update the database.

## Production Considerations

1. **Ollama Access**: For production, Ollama needs to be publicly accessible or switch to a hosted LLM like OpenAI/Anthropic
2. **Convex Pro**: Upgrade to Convex Pro for production workloads
3. **Environment**: Use proper environment variables for production URLs
